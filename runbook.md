# 週次デルタ収集 — Runbook

これは explore-engine のリサーチランナー指示書です。Claude Code に `claude -p "$(cat runbook.md)"` の形で食わせて実行します。

## ミッション

サービス起点（explore）で、**直近7日間に能力フロンティアが動いた事象**を全ジャンル横断で拾い、1本の markdown にまとめて `weekly/weekly_YYYY-MM-DD.md` に保存する（YYYY-MM-DD は今日の日付）。

## デフォルトジャンル（自分用に書き換えて使う）

1. AIコーディング / エージェント開発
2. エージェント基盤・自動化・オーケストレーション
3. 生成ビジュアル / インタラクティブ / メディアアート
4. アルゴ取引 / 金融×AI / 時系列
5. 個人開発フルスタック / エッジ / BaaS
6. UI/UX デザイン自動化
7. 科学計算 / 数値シミュレーション / 物理×AI
8. ナレッジ管理 / アイデア生成 / 生産性
9. モデルの新能力（横断）

## 出力フォーマット

ファイル先頭：

```
# 週次デルタ YYYY-MM-DD（YYYY-MM-DD 〜 YYYY-MM-DD）

直近7日間に天井を動かしたものだけ。動きが薄いジャンルは「動きなし」と1行書く。
```

各エントリ：

```
## N. タイトル　〔時期: YYYY-MM-DD〕〔ジャンル: XX〕
**規模感**: 大/中/小 — 理由
**能力**: 何が新たに可能になったか
**可能性空間**: 一般化したクラス
**before → after**: 過去の天井 → 現在の天井
**出典**: https://...
```

規模感（大→中→小）の順に並べる。

## やってはいけないこと

- 「君はXすべき」「あなたに向いてる」など**個人への決め打ち**
- 製品AとBの**ピア比較**（時間に対する天井移動だけが対象）
- **1週間より古い**情報（直近7日に限定）

## 並列化（任意）

ジャンルごとに研究サブエージェントを並列発射してから1本に統合してよい。同時 ≤ 7 並列を目安に。

## 出力ルール

- 保存先: `weekly/weekly_$(date +%Y-%m-%d).md`
- WSL運用なら必ず WSL ネイティブパスに書く（`/mnt/c` には直書きしない — Defender隔離）
- 動きが薄いジャンルは「動きなし」とだけ1行書く（無理に埋めない）

## 完了後

`./build.sh` を実行して `.html` と `index.html` を再生成すること。これはエンジンの一部であり、生成自体を完了とみなさない。
